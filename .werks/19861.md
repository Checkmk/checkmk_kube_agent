# Fix steadily increasing memory usage of machine sections node collector

key        | value
---------- | ---
date       | 2026-04-29T12:12:39.863577+00:00
version    | 2.0.0-alpha.1
class      | fix
edition    | community
component  | node-collector
level      | 1
compatible | yes

The machine sections node collector pod's memory usage grew steadily across
agent invocations, eventually sawtoothing against the configured memory
limit. The growth was caused by Python interpreter invocations during agent
startup leaving stale entries in the kernel's filesystem cache (counted against
the pod's cgroup). We now set PYTHONDONTWRITEBYTECODE=1 in the DaemonSet, which
eliminates the growth. After upgrade, the pod's reported memory usage will be
lower and stable.
